\section{Individual Contributions}
% ============================================================

\textbf{Rahul Ranjan Sah (50\%)} designed and implemented Modules 1 and 2. For Module 1, he built the knowledge base architecture with ProbLog integration, implemented the 11-dimension compatibility scoring system, and coded the Krumhansl-Kessler key profiles, circle-of-fifths distance calculations, and Bhattacharyya distance for MFCC similarity. For Module 2, he developed the bidirectional beam search algorithm, created the MusicBrainzDB PostgreSQL client with connection pooling, and implemented the ListenBrainz and AcousticBrainz API clients. He also handled database setup and replication configuration, conducted performance benchmarking, wrote unit tests for both modules, and managed research review and citation gathering.

\textbf{Michael Thomas (50\%)} designed and implemented Modules 3 and 4 along with the user-facing components. For Module 3, he built the playlist assembly system with six constraint types and the min-conflicts solver, developed the three-level explanation system, implemented user preference learning using exponential moving average, and integrated the Essentia fallback for tracks without AcousticBrainz data. For Module 4, he created the feature engineering pipeline, trained the logistic regression and MLP classifiers, and built the training data pipeline with database integration. He also implemented the Flask REST API and developed the React frontend. He wrote unit tests for Modules 3 and 4.

Both team members participated in architecture design sessions, conducted code reviews via pull requests, performed integration testing collaboratively, and shared documentation responsibilities across modules.
